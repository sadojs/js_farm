## 2. 개선 영역 — A: UI/UX 미해결 이슈

### A-01: 빈 상태(Empty State) 시스템 구축
- **대상**: Dashboard, Sensors, Groups, Automation, Alerts 등 모든 목록 페이지
- **현황**: "데이터가 없습니다" 텍스트만 표시 (M-02, M-03)
- **개선안**:
  - SVG 일러스트레이션 + 설명 텍스트 + CTA 버튼 조합의 EmptyState 공통 컴포넌트
  - 페이지별 맞춤 메시지: "첫 장비를 등록해보세요", "센서를 그룹에 연결하세요" 등
  - 가이드 스텝 표시: 장비등록 → 그룹설정 → 센서설정 → 모니터링
- **우선순위**: HIGH
- **예상 효과**: 신규 사용자 온보딩 경험 대폭 향상

### A-02: 모바일 리포트 레이아웃 개선
- **대상**: Reports.vue 모바일 뷰 (M-04, mob-04)
- **현황**: 기간 버튼 세로 쌓임, 다운로드 라벨 잘림, 차트 가독성 부족
- **개선안**:
  - 기간 선택을 가로 스크롤 Chip 형태로 변경
  - 다운로드 버튼을 아이콘 전용으로 변경
  - 차트 전체 너비 확장 + 터치 데이터포인트 표시
  - 테이블 → 카드형 리스트 변환 (모바일 전용)
- **우선순위**: HIGH

### A-03: 장비 카드 디자인 통일
- **대상**: Devices.vue (m-02)
- **현황**: 장비 타입별 카드 디자인 불일치 (센서=보라, 개폐기=텍스트, 관수=태그)
- **개선안**:
  - 타입별 통일 카드 템플릿: 아이콘 + 이름 + 상태뱃지 + 액션
  - 타입별 액센트 컬러: 센서(보라), 환풍기(파랑), 개폐기(주황), 관수(청록)
  - 온라인/오프라인 상태 표시 통일
- **우선순위**: MEDIUM

### A-04: 모바일 터치 타겟 & 접근성 개선
- **대상**: 전체 모바일 UI (mob-02, A11y 55점)
- **현황**: 아이콘 버튼 터치 영역 < 44px, tooltip 부재, 스크린리더 지원 부족
- **개선안**:
  - 모든 인터랙티브 요소 최소 44x44px 터치 타겟 보장
  - 아이콘 버튼에 aria-label + tooltip 추가 (m-03)
  - 폼 요소에 적절한 label 연결
  - 키보드 네비게이션 지원 (tab order, focus visible)
- **우선순위**: MEDIUM-HIGH

### A-05: 환경 모니터링 접이식 UI
- **대상**: Sensors.vue 모바일 뷰 (mob-03)
- **현황**: 스크롤 3868px, 모든 섹션 펼쳐진 상태
- **개선안**:
  - 종합 점수(46점) 상단 고정 + 탭 네비게이션 (내부환경/외부환경/종합평가)
  - 개별 센서 항목을 Collapsible 아코디언으로 변경
  - 현재값 + 상태뱃지만 요약 표시, 탭하면 상세 차트 펼침
- **우선순위**: MEDIUM

### A-06: 사이드바 하단 활용
- **대상**: 전체 레이아웃 사이드바 (M-01)
- **현황**: 메뉴 항목 아래 빈 공간
- **개선안**:
  - 시스템 상태 미니 위젯: 온라인 장비 수, 활성 알림 수
  - 빠른 액션: "장비 동기화", "데이터 새로고침"
  - 또는 최근 알림 미리보기
- **우선순위**: LOW

## 3. 개선 영역 — B: 신규 기능 아이디어

### B-01: 대시보드 위젯 커스터마이징
- **설명**: 사용자가 대시보드 위젯을 드래그&드롭으로 재배치하고, 필요한 위젯만 선택 표시
- **위젯 후보**: 날씨, 장비 상태, 센서 요약, 자동화 로그, 수확 일정, 알림 피드
- **기대효과**: 역할별(관리자 vs 농장주) 맞춤 대시보드
- **난이도**: MEDIUM-HIGH
- **추천도**: ★★★☆☆ (현재 대시보드가 충분히 기능적이므로 후순위)

### B-02: 알림 센터 + 푸시 알림
- **설명**:
  - 앱 내 알림 센터(Bell icon → 드롭다운)에서 모든 알림 통합 관리
  - 센서 임계값 초과, 장비 오프라인, 자동화 실행 결과 등
  - 브라우저 Push Notification 또는 카카오톡 알림 연동
- **기대효과**: 실시간 대응력 향상, 현장 불필요 방문 감소
- **난이도**: MEDIUM
- **추천도**: ★★★★★ (IoT 플랫폼의 핵심 기능)

### B-03: 자동화 실행 로그 & 히스토리
- **설명**:
  - 자동화 룰이 언제 실행되었는지, 어떤 조건이 트리거되었는지 타임라인 표시
  - 실행 성공/실패 상태, 실행 소요 시간
  - 특정 기간의 자동화 실행 통계 차트
- **기대효과**: 자동화 신뢰도 확인, 룰 최적화 근거 제공
- **난이도**: MEDIUM
- **추천도**: ★★★★★ (자동화 기능의 투명성 강화)

### B-04: 센서 데이터 비교 뷰
- **설명**:
  - 여러 센서/그룹의 데이터를 하나의 차트에 오버레이하여 비교
  - 예: 1동 vs 2동 온도 비교, 내부 vs 외부 습도 비교
  - 기간별 트렌드 비교 (이번 주 vs 지난 주)
- **기대효과**: 데이터 기반 의사결정 지원
- **난이도**: MEDIUM
- **추천도**: ★★★★☆ (리포트 페이지 확장)

### B-05: 모바일 PWA 최적화
- **설명**:
  - manifest.json + Service Worker로 PWA 설치 지원
  - 오프라인 기본 화면 (마지막 동기화 데이터 캐싱)
  - 홈 화면 추가 시 네이티브 앱 느낌
  - 스플래시 스크린, 앱 아이콘
- **기대효과**: 모바일 접근성 극대화, 농장 현장 사용 편의
- **난이도**: LOW-MEDIUM
- **추천도**: ★★★★★ (농장 현장에서 주로 모바일 사용)

### B-06: 온보딩 투어 / 가이드 모드
- **설명**:
  - 첫 로그인 시 또는 수동 실행으로 플랫폼 주요 기능 투어
  - 스텝별 하이라이트 + 말풍선 안내 (예: Shepherd.js)
  - "1. 장비 등록 → 2. 그룹 생성 → 3. 자동화 설정" 순서 안내
- **기대효과**: 신규 사용자 학습 곡선 감소
- **난이도**: MEDIUM
- **추천도**: ★★★★☆ (A-01 빈 상태와 연계)

### B-07: 다국어(i18n) 지원 기반 마련
- **설명**:
  - vue-i18n 도입, 한국어 기본 + 영어 지원
  - 현재 하드코딩된 한국어 문자열을 키 기반으로 전환
  - 자동화 동작 "open" → "열기" 같은 동적 번역 (m-07)
- **기대효과**: 해외 진출 기반, 코드 품질 향상
- **난이도**: HIGH (전체 문자열 추출 필요)
- **추천도**: ★★★☆☆ (당장 필요하지는 않으나 장기적 가치)

### B-08: 데이터 내보내기 강화
- **설명**:
  - 리포트 데이터 CSV/Excel 내보내기 (현재 부분 지원)
  - 센서 데이터 기간별 벌크 다운로드
  - 자동화 로그 내보내기
  - PDF 리포트 생성 (주간/월간 요약)
- **기대효과**: 외부 보고, 데이터 분석 활용
- **난이도**: MEDIUM
- **추천도**: ★★★★☆

## 6. 기술적 고려사항

### 공통 컴포넌트 신규 생성 필요
- `EmptyState.vue` — 빈 상태 표시 공통 컴포넌트

- `SensorCompareChart.vue` — 센서 데이터 비교 차트

### 백엔드 변경 필요 (B-02, B-03)
- `notification` 모듈: WebSocket 실시간 알림 + 알림 저장/읽음 처리
- `automation-log` 모듈: 자동화 실행 이력 저장 및 조회 API
- Push Notification: Web Push API 또는 FCM 연동

### 기존 코드 영향 범위
- Phase 1: 프론트엔드만 변경 (백엔드 무관)
- Phase 2: 프론트엔드 + 백엔드 모두 변경 필요
- Phase 3-4: 주로 프론트엔드, 일부 백엔드 API 추가

## 7. 특별 추천 — TOP 3 신규 기능

### 1위: 알림 센터 + 푸시 알림 (B-02) ★★★★★
> IoT 플랫폼의 가치는 "실시간 대응"에 있음. 센서 이상, 장비 오프라인 시 즉각 알림이 농장 운영의 핵심.

### 2위: 자동화 실행 로그 (B-03) ★★★★★
> "룰이 제대로 동작하고 있는지" 확인할 수 없으면 자동화에 대한 신뢰가 형성되지 않음. 투명한 실행 기록이 필수.

### 3위: PWA 최적화 (B-05) ★★★★★
> 농장 현장에서는 PC보다 스마트폰 사용이 압도적. 앱 설치 없이 홈 화면에 추가하는 PWA가 가장 실용적.
