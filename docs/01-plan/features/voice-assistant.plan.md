# voice-assistant Plan

> 웹 내장 음성 어시스턴트 — 마이크 버튼으로 음성 명령, 음성 응답으로 장치 제어/정보 조회

## 1. 배경 및 문제점

### 1.1 현재 상태
- 장치 제어·날씨·상태 조회가 웹 UI 탐색(여러 페이지 이동)을 통해서만 가능
- 농장 현장에서 손이 더럽거나 바쁠 때 터치 조작이 불편
- 외부 메신저(텔레그램 등) 없이 웹 앱 내에서 직접 음성 제어를 원함

### 1.2 요구사항 요약
- 웹 앱에 **플로팅 마이크 버튼** 추가 → 누르면 실시간 음성 인식
- 음성 명령에 따라 장치 제어, 자동화 관리, 날씨/상태 조회
- 결과를 **음성(TTS)으로 읽어주고** + 텍스트로도 표시
- **완전 무과금**: 브라우저 내장 Web Speech API 사용 (서버 비용 없음)

## 2. 핵심 기술: Web Speech API

### 2.1 음성 인식 (STT) — SpeechRecognition
- 브라우저 내장, **무료**, 서버 불필요
- 한국어 지원: `lang = 'ko-KR'`
- Chrome, Edge, Safari 지원 (사용자 기기: iPhone 15 Pro Safari ✅)
- PWA에서도 동작

### 2.2 음성 합성 (TTS) — SpeechSynthesis
- 브라우저 내장, **무료**
- 한국어 음성 지원: iOS/macOS/Chrome 모두 한국어 TTS 내장
- `speechSynthesis.speak(utterance)` 한 줄로 동작

### 2.3 비용: 0원
| 항목 | 방식 | 비용 |
|------|------|------|
| 음성 인식 (STT) | Web SpeechRecognition API | **무료** |
| 음성 합성 (TTS) | Web SpeechSynthesis API | **무료** |
| 명령 파싱 | 규칙 기반 (프론트엔드) | **무료** |
| 장치 제어 | 기존 REST API 호출 | **추가 비용 없음** |

## 4. 기능 요구사항 (FR)

### FR-01: 플로팅 마이크 버튼
- 모든 페이지에 표시되는 플로팅 버튼 (App.vue 레벨)
- 위치: 우측 하단, BottomTabBar 위
- 상태: 대기(회색) → 듣는 중(빨강 펄스) → 처리 중(로딩) → 응답(초록)
- 탭: 음성 인식 시작/중지 토글

### FR-02: 음성 인식 (STT)
- Web SpeechRecognition API 사용
- `lang: 'ko-KR'` (한국어)
- 실시간 중간 결과 표시 (interimResults)
- 인식 완료 시 자동으로 명령 파서에 전달
- 마이크 권한 요청 처리

### FR-03: 명령 파서 (프론트엔드)
- 텔레그램 봇과 동일한 규칙 기반 패턴 매칭
- 장치 제어: "팬 켜줘", "1동 관수 시작", "개폐기 열어"
- 정보 조회: "날씨 알려줘", "온도 몇 도야", "장치 상태"
- 자동화: "자동화 목록", "환기 룰 켜줘"
- 매칭 실패: "명령을 이해하지 못했어요. 다시 말씀해주세요."

### FR-04: 명령 실행 (기존 API 활용)
- 장치 제어: `deviceApi.control(id, commands)` — 기존 API 그대로 사용
- 날씨 조회: `dashboardApi.getWeather()` — 기존 API 그대로 사용
- 장치 목록: `deviceApi.getAll()` — store에서 캐시된 데이터 활용
- 자동화: `automationApi.toggle()`, `automationApi.runNow()` — 기존 API
- **백엔드 변경 없음** — 프론트엔드만으로 완성

### FR-05: 음성 응답 (TTS)
- Web SpeechSynthesis API 사용
- 한국어 음성으로 결과 읽어주기
- 예: "1동 팬을 켰습니다", "현재 온도 18.5도, 습도 65%입니다"
- TTS 중 마이크 비활성화 (에코 방지)

### FR-06: 대화 패널 UI
- 하단 슬라이드업 패널 (마이크 버튼 탭 시 표시)
- 대화 기록 표시 (사용자 → 시스템 응답)
- 최근 5개 대화 유지
- 패널 밖 탭 또는 X 버튼으로 닫기

### FR-07: 텍스트 입력 모드 (폴백)
- 음성 인식 불가 환경(Firefox 등)에서 텍스트 입력 필드 표시
- 동일한 명령 파서 사용
- 키보드 Enter로 전송

## 5. 비기능 요구사항

### NFR-01: 브라우저 호환성
- 필수 지원: Chrome, Safari (iOS/macOS), Edge
- 미지원 시: 텍스트 입력 모드로 폴백
- iPhone 15 Pro Safari에서 최우선 테스트

### NFR-02: 성능
- 음성 인식 → 명령 실행 → 응답: < 3초
- 플로팅 버튼이 기존 페이지 성능에 영향 없음

### NFR-03: 모바일 UX
- 플로팅 버튼: BottomTabBar와 겹치지 않는 위치
- 대화 패널: 화면 하단 50% 이하 차지
- 터치 영역: 최소 44x44px

## 6. 구현 범위

### 백엔드: 변경 없음
- 기존 API 그대로 사용 (장치 제어, 날씨, 자동화)
- 새 엔드포인트 불필요

### 프론트엔드: 새 파일 3~4개
| 파일 | 역할 |
|------|------|
| `composables/useVoiceAssistant.ts` | SpeechRecognition + SpeechSynthesis + 명령 파서 |
| `components/common/VoiceAssistant.vue` | 플로팅 버튼 + 대화 패널 UI |
| `stores/voice.store.ts` (선택) | 대화 기록 상태 관리 (필요시) |

### 수정 파일: 1개
| 파일 | 변경 |
|------|------|
| `App.vue` | VoiceAssistant 컴포넌트 추가 (인증된 사용자에게만) |

## 7. 명령어 레퍼런스

| 음성 명령 | 동작 | 응답 예시 |
|----------|------|----------|
| "팬 켜줘" | 팬 장치 ON | "1동 팬을 켰습니다" |
| "1동 팬 꺼" | 1동 팬 OFF | "1동 팬을 껐습니다" |
| "관수 시작" | 관수 ON | "관수를 시작했습니다" |
| "개폐기 열어" | 개폐기 열림 | "1동 개폐기를 열었습니다" |
| "날씨 알려줘" | 날씨 조회 | "현재 온도 18.5도, 습도 65%입니다" |
| "장치 상태" | 장치 목록 | "팬 1개 켜짐, 관수 꺼짐, 개폐기 닫힘" |
| "자동화 목록" | 룰 목록 | "환기 룰 활성, 관수 룰 비활성" |
| "환기 룰 켜줘" | 자동화 토글 | "환기 룰을 활성화했습니다" |

## 8. 구현 우선순위

| 단계 | 기능 | 우선순위 |
|------|------|----------|
| **Phase 1** | 플로팅 버튼 + 음성 인식 + 장치 제어 | P0 |
| **Phase 2** | 음성 응답(TTS) + 대화 패널 | P0 |
| **Phase 3** | 날씨/상태 조회 + 자동화 관리 | P1 |

| 항목 | 텔레그램 봇 | 웹 음성 어시스턴트 |
